# 稳定性验收测试指南（test1）

适用对象：当前 `asdun_access` 版本（已可在 VNC 显示并进行人脸识别与情绪检测）。  
测试目标：先确认“系统稳定可用”，再进入模型精度优化阶段。

## 1. 验收指标（必须项）

| 指标ID | 指标名称 | 验收目标 | 通过判定 |
|---|---|---|---|
| S1 | 启动成功率 | 连续启动 10 次，成功 10 次 | 无崩溃、无黑屏卡死 |
| S2 | 基础功能完整性 | 录入、覆盖、识别、情绪、退出均可操作 | 关键流程全部可走通 |
| S3 | 连续运行稳定性 | 单次连续运行 >= 60 分钟 | 无崩溃、无明显卡死 |
| S4 | 识别实时性 | 主观流畅，显示刷新不明显掉帧 | 现场体验可接受（门禁场景） |
| S5 | 资源稳定性 | CPU/内存无持续失控增长 | 内存无明显泄漏趋势 |
| S6 | 录入质量门控有效性 | 无脸/模糊/过小脸不能抓拍 | 仅 `ready` 状态可按 `s` 成功 |
| S7 | 录入数量约束 | 单人录入 2 张后自动返回菜单 | 行为一致无误 |
| S8 | 同名覆盖保护 | 输入已存在姓名时触发 `y/n` 确认 | `n` 不改数据，`y` 覆盖重录 |
| S9 | 重启持久化 | 程序重启后仍可识别已录入人员 | 数据库与图片可复用 |
| S10 | Unknown 判定 | 未录入人员显示 `Unknown` | 不应误显示为已知人员 |
| S11 | 情绪结果可读性 | 框上有情绪标签和置信度 | 标签刷新可观察、非全程固定 |
| S12 | 人机交互可恢复 | `q` 退出识别、异常后可重启恢复 | 不需重启系统即可继续测试 |

## 2. 测试前准备

- 在树莓派 VNC 桌面中打开终端，进入项目目录：
```bash
cd /home/asdun/asdun_pi
```
- 确认可执行文件存在：
```bash
ls -l ./build_rpi/asdun_access
```
- 建议开启一个“监控终端”（可选）：
```bash
watch -n 2 "ps -o pid,%cpu,%mem,rss,etime,cmd -C asdun_access"
```
- 建议开启温度监控（可选）：
```bash
watch -n 5 vcgencmd measure_temp
```

## 3. 测试步骤（按顺序执行）

### T01 启动/退出稳定性（S1, S12）

1. 运行程序：`./build_rpi/asdun_access`
2. 进入识别模式后按 `q` 返回菜单，再按 `0` 退出。
3. 重复 10 次。

记录：
- 是否出现崩溃、卡死、窗口打不开。
- 是否每次都能正常退出并再次启动。

### T02 录入质量门控（S6）

1. 进入录入模式，输入新姓名（如 `user_a`）。
2. 分别测试三种情况：
- 画面无脸时按 `s`。
- 模糊画面（快速晃动）时按 `s`。
- 人脸很小（远离镜头）时按 `s`。
3. 最后保持清晰且人脸足够大，直到状态 `ready` 后按 `s`。

记录：
- 前三种情况应不能成功录入。
- `ready` 时应可成功录入。

### T03 录入2张自动返回（S7）

1. 延续 T02 同一用户，抓拍到第 2 张有效样本。
2. 观察是否自动返回主菜单。

记录：
- 是否严格在第 2 张后返回。

### T04 同名覆盖保护（S8）

1. 再次录入同名 `user_a`。
2. 先输入 `n`，确认不覆盖并返回菜单。
3. 再次同名录入，这次输入 `y`，完成重新录入 2 张。

记录：
- `n` 后旧数据应保留。
- `y` 后应进入重录流程并可正常识别。

### T05 已知/未知识别（S10）

1. 进入识别模式，先用已录入人员对准镜头。
2. 再换未录入人员。

记录：
- 已录入人员应显示姓名（绿框）。
- 未录入人员应显示 `Unknown`（红框）。

### T06 情绪显示稳定性（S11）

1. 已录入人员在识别模式下做几种明显表情变化（中性/微笑/严肃）。
2. 观察标签是否更新，是否频繁抖动。

记录：
- 是否存在合理变化。
- 是否出现每帧乱跳（若有，后续调 EMA/阈值）。

### T07 重启持久化（S9）

1. 退出程序。
2. 重新运行程序进入识别模式。
3. 验证已录入人员无需重新录入即可被识别。

记录：
- 持久化是否有效。

### T08 连续运行 60 分钟（S3, S4, S5）

1. 保持识别模式连续运行 >= 60 分钟。
2. 每 10 分钟记录一次 CPU、内存、温度、主观流畅度。

记录：
- 是否崩溃。
- 内存是否持续单向上涨（疑似泄漏）。
- 是否出现明显卡顿恶化。

## 5. 最终通过标准

- T01-T08 全部 PASS。
- 没有阻断级问题（崩溃、无法启动、录入不可用、识别全失效）。
- 连续 60 分钟内无不可恢复故障。
- 能稳定完成“录入 -> 识别 -> 重启后复用”闭环。

## 6. 若失败，优先排查顺序

1. 先看是否是显示环境问题（VNC/Qt/OpenCV 窗口）。  
2. 再看摄像头链路是否稳定（`camera_source`、分辨率、占用冲突）。  
3. 再看参数是否过激（`detect_interval`、`emotion_interval`、`match_threshold`）。  
4. 最后看模型与代码逻辑（跟踪抖动、置信度映射、持久化一致性）。

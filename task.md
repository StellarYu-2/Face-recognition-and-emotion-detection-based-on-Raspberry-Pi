# 核心功能需求
## 1. 录入与抓拍功能 (Enrollment)
- 终端交互：启动程序进入主控状态机菜单，选择录入模式后，提示输入人员姓名（如 zhangsan）。
- 画面校验（质量过滤）：只有在实时画面中检测到有效人脸（需包含基础质量判断：如 Bounding Box 尺寸足够大、基于拉普拉斯方差的模糊度检测达标），终端才允许响应按键。
- 抓拍逻辑：按下 `s` 键抓拍，存入 `./data/images/{name}/`，同时提取人脸特征向量（Embeddings）并序列化保存到本地硬盘（持久化，如 bin、npy 或 SQLite），避免重启后重复计算。
- 限制与退出：单人拍满 2 张有效人脸后自动返回主菜单。

## 2. 录入冲突与覆盖机制 (Overwrite Protection)
- 在输入姓名后，检查本地持久化存储中是否已有该人员。
- 若已有，终端拦截并提示：“该人员已有记录，是否清空旧数据并重新录入？(y/n)”。输入 `y` 则清理旧图片和特征，`n` 则返回主菜单。

## 3. 实时识别与性能压榨 (Real-time Recognition - 核心难点)
- 极致并发：树莓派算力有限，必须使用多线程/多进程分离“视频流读取（Producer）”和“推理渲染（Consumer）”，避免 I/O 阻塞推理。
- libcamera 适配：请使用最高效的取帧方式（若 Python 用 Picamera2；若 C++ 建议 libcamera API 或 OpenCV 配合 GStreamer 管道）。
- 跳帧检测与跟踪：为保障画面流畅度，请实现跳帧策略（如每 N 帧做一次完整的人脸检测+特征提取识别，中间帧仅使用轻量级的简单跟踪器，如基于 IoU 的位置匹配或简易卡尔曼滤波）来更新 Bounding Box。
- UI 展示：匹配成功用绿色框及姓名标注；失败用红色框标注 `Unknown`。

## 4. 识别置信度显示 (Confidence Score Mapping)
- 框上方需附带置信度百分比（如 `zhangsan (85.3%)`）。
- 数学要求：底层模型通常返回 Distance（欧氏距离或余弦相似度），请编写一个符合数学逻辑的非线性（如 Sigmoid 变体）或线性转换函数，将其动态映射到 0-100%。
- **请务必使用 LaTeX 公式（使用 `$` 或 `$$` 包裹）在注释或解释中给出推导过程和阈值（Threshold）设定的依据。**

## 5. 情绪识别模式
- 进入该模式之后，我将情绪分成开心，伤心，愤怒，平静四种，摄像机可以拍摄人的情绪并记录其的情绪
- 具体例子是：比如看到张三开心，就存放张三开心的记录置于存储库里面，便于我后续开发导出其表情记录日志：即记录了开心的占比，以及各种情绪的占比，便于后续我的管理与应用，因为我后续想做的是可以导出一周心情占比的功能
